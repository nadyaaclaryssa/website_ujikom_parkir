==================================================
MATERI PRESENTASI UKK - APLIKASI PARKIR (PARLINE)
==================================================

SLIDE 1: JUDUL DOKUMEN / COVER
==================================================
• Judul Projek: PARLINE (Sistem Manajemen Parkir Cerdas & Terintegrasi)
• Deskripsi: Aplikasi web komprehensif untuk mendigitalisasi area parkir (Pencatatan Real-time, Kalkulasi Otomatis, & Laporan Pendapatan).
• Teknologi Inti: PHP Native (8.x), MySQLi, HTML5, CSS3.
• Kelas / Jurusan: Rekayasa Perangkat Lunak (RPL)

==================================================
SLIDE 2: LATAR BELAKANG & TUJUAN
==================================================
Latar Belakang Masalah: 
1. Pencatatan manual menggunakan kertas sangat rentan hilang dan rawan manipulasi.
2. Sering terjadi miskomunikasi antara petugas dan pengendara mengenai sisa slot parkir kosong.
3. Kesalahan hitung tarif parkir manual oleh petugas loket berujung hilangnya omset.

Solusi PARLINE (Tujuan Aplikasi):
1. Mendigitalisasi gerbang parkir dengan sistem Check-In & Check-Out cepat terkomputerisasi.
2. Auto-Kalkulasi tarif berdasarkan argo durasi (jam) yang 100% akurat.
3. Menyediakan dashboard pantauan slot parkir real-time (ketersediaan lahan).
4. Transparansi laporan harian + Grafik visual bagi pemangku kepentingan (Owner).

==================================================
SLIDE 3: STRUKTUR DATABASE RELASIONAL (7 TABEL)
==================================================
Arsitektur database dibangun efisien dan ternormalisasi menggunakan 7 tabel aktif:
1. tb_user: Menyimpan data kredensial akses (Admin, Petugas, Owner) beserta Password (MD5).
2. tb_tarif: Parameter tarif dasar & per-jam setiap jenis kendaraan (Mobil, Motor).
3. tb_area_parkir: Pengendali kapasitas slot fisik (Lantai 1, Blok A, dsb) secara dinamis.
4. tb_kendaraan: Entitas fisik kendaraan pelanggan (Plat, Warna, Jenis).
5. tb_transaksi: Jantung sistem, merekam durasi, biaya, dan riwayat "Masuk/Keluar".
6. tb_log_aktivitas: Sistem Audit Trail (Catatan interaksi CRUD user di aplikasi).
7. tb_log: (Opsional/Legacy) Catatan legacy aktivitas petugas.

==================================================
SLIDE 4: HAK AKSES SISTEM (ROLE-BASED ACCESS)
==================================================
Aplikasi mengusung 3 pilar batas otoritas utama:
1. ADMIN (Pengendali Sistem):
   -> Punya akses penuh (*God Mode*).
   -> Kelola Master Data: Tambah/Edit/Hapus User Karyawan, Setting Tarif Harga, dan Buka Area Parkir Baru.
   -> Pantau Log Audit keamanan.

2. PETUGAS (Garda Operasional):
   -> Fokus di loket antarmuka Input. 
   -> Melakukan Check-In (Pilih Area & Plat), Check-Out (Kalkulator Tagihan), dan Print Struk.
   -> Layar dihiasi "Live HUD" penunjuk sisa kuota parkir.

3. OWNER (Pemangku Kepentingan):
   -> Visibilitas penuh ke Finansial tanpa bisa merusak data (View-Only).
   -> Dashboard Chart/Grafik Tren Omset 7 hari terakhir.
   -> Tarik & Ekspor laporan spesifik dengan filter rentang tanggal.

==================================================
SLIDE 5: TEKNOLOGI & KEAMANAN SISTEM
==================================================
• Backend (PHP & MySQLi): Logika yang efisien dengan implementasi *mysqli_real_escape_string* sebagai pelindung anti-hacker (SQL Injection).
• Keamanan Sesi (Session Start): Pengendali batas tembok api yang memaksa pengguna ter-redirect (lempar paksa) bila mencoba masuk halaman tanpa izin/login.
• Enkripsi (MD5 Algoritma): Semua sandi tersimpan sebagai kode acak tidak beraturan di server.
• Visual Layer: Menggunakan antarmuka Glass-morphism (Vanilla CSS) sehingga modern dan elegan tanpa memberatkan *loading* browser.

==================================================
SLIDE 6: ALUR SIKLUS HIDUP TRANSAKSI (WORKFLOW)
==================================================
[STEP 1 - CHECK IN]: 
Kendaraan masuk -> Petugas ketik Plat & Pilih Area -> Sistem menyimpan ke `tb_kendaraan` dan merekam jam kedatangan absolut ke `tb_transaksi` -> Slot `tb_area_parkir` langsung berkurang 1.

[STEP 2 - CHECK OUT]: 
Kendaraan keluar -> Petugas ketik plat -> Sistem menghitung selisih waktu (`Date::Diff`) antara jam kedatangan vs kepulangan -> Selisih jam dikali tarif -> Tagihan muncul.

[STEP 3 - FINISHING]: 
Petugas klik simpan -> Transaksi sukses dibayar -> Slot `tb_area_parkir` bertambah 1 kembali -> Print Struk Kertas Termal otomatis (Window Print UI).
